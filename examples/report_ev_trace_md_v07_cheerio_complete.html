CT.1 Adecuación al uso previsto declarado B Cumple El uso previsto de cheerio está documentado con claridad en múltiples fuentes públicas oficiales del proveedor. El README en npm (https://www.npmjs.com/package/cheerio) y el repositorio GitHub (https://github.com/cheeriojs/cheerio) describen la herramienta como 'la librería rápida, flexible y elegante para parsear y manipular HTML y XML', implementando un subconjunto de la API de jQuery sobre un modelo DOM server-side. La documentación oficial en cheerio.js.org detalla guías de uso, API completa y casos de uso (parsing, traversal, manipulación y renderizado de HTML/XML). La versión 1.1.2 es una versión estable post-1.0.0 con funcionalidades de parsing HTML/XML completamente documentadas. No se trata de un SaaS ni de una herramienta con componentes sanitarios directos: su uso previsto como biblioteca de parseo de HTML en pipelines de software queda inequívocamente definido en la documentación pública. Las funcionalidades (carga, selección CSS, traversal, manipulación del DOM, serialización) cubren los requisitos típicos de transformación/extracción de contenido HTML en aplicaciones Node.js. https://www.npmjs.com/package/cheerio | https://github.com/cheeriojs/cheerio | https://cheerio.js.org/ Comparar el uso previsto declarado por el proveedor con el uso interno.

CS1.1 Procedencia y canal de distribucion oficial B Cumple parcialmente cheerio 1.1.2 está publicado en el canal oficial registry.npmjs.org (https://registry.npmjs.org/cheerio/-/cheerio-1.1.2.tgz) y el registro npm proporciona shasum (SHA-1) e integrity (SHA-512 SRI) para cada versión en sus metadatos dist, verificables mediante 'npm view cheerio@1.1.2 dist.integrity dist.shasum'. Sin embargo, la versión 1.1.2 fue publicada sin attestation de provenance Sigstore/SLSA: esta funcionalidad solo está disponible para paquetes publicados con --provenance flag desde npm CLI 9.5+ (GA octubre 2023), y la versión evaluada es anterior a esa práctica. No existe firma GPG/Sigstore vinculada al artefacto 1.1.2 en el registro. El riesgo residual es mitigable organizativamente fijando el hash SHA-512 en package-lock.json y ejecutando 'npm audit signatures' para verificar la firma de registro (ECDSA) que sí aplica a todos los paquetes del registry. https://registry.npmjs.org/cheerio (packument oficial con campos dist.integrity y dist.shasum); https://docs.npmjs.com/generating-provenance-statements/ (documentación provenance Sigstore npm); https://github.com/cheeriojs/cheerio (repositorio oficial cheeriojs) Registrar el canal de descarga utilizado en el expediente.
CS1.2 Actividad y mantenibilidad del componente B/C Cumple parcialmente El proyecto cheerio está activo: la versión 1.0.0 se publicó en agosto de 2024 (cheerio.js.org/blog/cheerio-1.0/) y la última versión en npm es la 1.2.0, publicada hace aproximadamente 5 meses (npmjs.com/package/cheerio). Snyk confirma cadencia de releases positiva y mantenimiento 'Healthy' con actividad de PR/issues en el repositorio. GitHub muestra commits de dependabot activos hasta enero 2026. Sin embargo, la versión evaluada (1.1.2) NO es la versión más reciente publicada (la latest es 1.2.0), lo que supone un riesgo de que la organización esté usando una versión que puede no incorporar las últimas correcciones. La actividad del proyecto en sí misma es sana, pero la versión específica desplegada queda desfasada respecto al latest, lo que genera una duda real de mantenibilidad para esa versión concreta. https://www.npmjs.com/package/cheerio — https://snyk.io/advisor/npm-package/cheerio — https://github.com/cheeriojs/cheerio/releases — https://cheerio.js.org/blog/cheerio-1.0/ Evaluar alternativas si no hay actividad reciente.
CS1.3 Vulnerabilidades conocidas (SCA) B No cumple cheerio 1.1.2 no presenta CVEs directos conocidos (Snyk confirma 'no direct vulnerabilities'). Sin embargo, la versión 1.1.2 incorpora undici como dependencia directa, paquete con múltiples CVEs activos con severidades high/medium: CVE-2026-6733 (HTTP response queue poisoning), CVE-2026-6734 (SOCKS5 connection pool reuse, credenciales a origen erróneo), CVE-2026-1527 (CRLF injection), CVE-2025-22150 (Insecure Randomness), CVE-2026-12151 y CVE-2026-9675 (DoS WebSocket). Los parches exigen undici ≥7.28.0/8.2.0, pero cheerio 1.1.2 depende de undici ^7.x sin garantía de resolución a versión parcheada según lockfile. Adicionalmente, 1.1.2 no es la versión latest (1.2.0), lo que implica posibles correcciones de dependencias no incluidas. No existe mitigación organizativa suficiente para vulnerabilidades de dependencia transitiva sin actualizar la versión del paquete. https://security.snyk.io/package/npm/cheerio (Snyk, sin CVEs directos); https://nvd.nist.gov (CVE-2026-6733, CVE-2026-6734 undici); https://security.snyk.io/vuln/SNYK-JS-UNDICI-8641354 (CVE-2025-22150 undici); https://advisories.gitlab.com/pkg/npm/undici/CVE-2026-1527/ (CVE-2026-1527 undici); https://securityonline.info/undici-vulnerabilities/ (CVE-2026-12151, CVE-2026-9675 undici) Ejecutar analisis SCA (Dependabot, OWASP Dependency-Check, Snyk).

La carencia de SBOM publicado y de política estructurada de parches no puede compensarse con medidas organizativas propias. https://github.com/cheeriojs/cheerio/blob/main/SECURITY.md | https://github.com/cheeriojs/cheerio/releases | https://www.npmjs.com/package/cheerio Generar o mantener internamente el SBOM del producto.
CS1.6 Integridad del artefacto descargado B Cumple parcialmente cheerio 1.1.2 está publicado en el registro oficial npm (registry.npmjs.org). El registro publica un campo 'dist.integrity' con hash SHA-512 (no SHA-256) y una firma ECDSA del registro verificable mediante 'npm audit signatures', lo que permite comprobar integridad del tarball frente a manipulación en tránsito. Sin embargo, no se encuentra evidencia pública de que cheerio 1.1.2 haya sido publicado con atestación de procedencia Sigstore/SLSA (el badge de provenance no aparece en la ficha npmjs.com del paquete), por lo que la cadena de custodia build→publicación no está criptográficamente atestada. El criterio solicita SHA-256, pero npm publica SHA-512; la verificación del hash es posible aunque en un algoritmo distinto al especificado. Mitigación organizativa: fijar el hash SHA-512 en package-lock.json y ejecutar 'npm audit signatures' en el pipeline CI. https://registry.npmjs.org/cheerio (dist.integrity / dist.signatures); https://docs.npmjs.com/verifying-registry-signatures/; https://www.npmjs.com/package/cheerio (ausencia de badge de provenance); https://github.com/cheeriojs/cheerio/releases Incorporar la verificación de integridad al pipeline CI/CD.

La única obligación es mantener el aviso de copyright en las copias distribuidas del software. No se detecta ningún componente de licencia restrictiva (GPL, AGPL, EUPL) en la licencia del paquete principal. La compatibilidad con la licencia del SaMD debe verificarse también respecto a las dependencias transitivas de cheerio (parse5, htmlparser2, etc.), tarea de naturaleza organizativa que queda fuera del alcance del proveedor. https://github.com/cheeriojs/cheerio/blob/main/LICENSE | https://github.com/cheeriojs/cheerio/blob/main/package.json | https://www.npmjs.com/package/cheerio | https://cdnjs.com/libraries/cheerio Mantener inventario de licencias. CS1.8 Fijacion de versión y control de cambios (pinning) B No verificable El criterio CS1.8 evalúa si la dependencia está fijada a una versión concreta en el fichero de definición del proyecto que la consume (package.json, package-lock.json, yarn.lock, etc.). Esto es una decisión de configuración de la organización usuaria, no del proveedor de la librería. Desde el lado del proveedor, se confirma que la versión 1.1.2 existe como artefacto inmutable e identificable en el registro npm de la URL oficial indicada (https://registry.npmjs.org/cheerio/-/cheerio-1.1.2.tgz). El registro npm permite referenciar versiones exactas sin rango (p.ej. "cheerio": "1.1.2" sin prefijos ^ ni ~). Sin embargo, si el pinning está realmente implementado en el fichero de definición del proyecto del fabricante es algo que solo puede verificarse inspeccionando los ficheros internos del proyecto (package.json / lockfile), los cuales no son públicos. La versión 1.2.0 es actualmente la latest según npmjs.com, lo que confirma que 1.1.2 es una versión anterior soportada pero no la actual. https://www.npmjs.com/package/cheerio (versión latest 1.2.0 confirmada); https://registry.npmjs.org/cheerio/-/cheerio-1.1.2.tgz (artefacto de versión exacta disponible en registro oficial) Establecer política de fijacion de versión. Cualquier cambio de versión reinicia la evaluación.
CS1.9 Riesgo de discontinuidad del componente C/O Cumple parcialmente La versión evaluada (1.1.2) está desactualizada y sin soporte activo: la versión actual en npm es 1.2.0 (publicada hace ~5 meses) y la 1.0.0 se lanzó en agosto de 2024, introduciendo breaking changes respecto a la rama 1.0.0-rc usada como base de 1.1.2. El repositorio cheeriojs/cheerio muestra actividad reciente (último commit el 26 jun 2026) con 30.395 estrellas y ~37,6 millones de descargas semanales, lo que descarta riesgo de abandono del proyecto en general. El mantenimiento recae principalmente en un único mantenedor activo (@fb55/Felix Boehm), con soporte de patrocinadores vía GitHub Sponsors y Open Collective, pero sin patrocinadores corporativos de primer nivel verificables públicamente. El riesgo real para la organización no es el abandono del proyecto sino el uso de una versión obsoleta (1.1.2 frente a 1.2.0 estable), que acumula más de 20.000 paquetes dependientes del canal actual. Se recomienda actualizar a 1.2.0 y monitorizar la continuidad del mantenedor principal. https://www.npmjs.com/package/cheerio (versión 1.2.0, 5 meses); https://github.com/cheeriojs/cheerio (commit Jun 26 2026); https://security.snyk.io/package/npm/cheerio (37.658.036 descargas/semana, mantenimiento 'Healthy'); https://github.com/sponsors/cheeriojs; https://opencollective.com/cheerio Para dependencias criticas con alto riesgo de abandono, evaluar alternativas.
CS1.10 Comportamiento en ejecución y acceso a datos del entorno C/O Cumple La evidencia pública disponible indica que cheerio 1.1.2 es una librería de parseo y manipulación de HTML/XML de propósito específico. Según la documentación oficial y el repositorio GitHub (cheeriojs/cheerio), el paquete opera exclusivamente sobre estructuras de datos DOM en memoria: 'no produce visual rendering, apply CSS, load external resources, or execute JavaScript'. No se ha encontrado evidencia de acceso a variables de entorno (process.env), credenciales del proceso, red, ni sistema de ficheros de forma autónoma. Snyk y Socket.dev no reportan vulnerabilidades conocidas de tipo exfiltración de datos para el paquete cheerio legítimo. No existen postinstall scripts maliciosos documentados en la versión 1.1.2. Los incidentes npm de tipo Shai-Hulud (sept. 2025) y otros analizados afectaron a paquetes distintos (chalk, debug, tinycolor, etc.), no a cheerio. La versión evaluada (1.1.2) no coincide con la versión latest (1.2.0), por lo que se recomienda verificar el hash del tarball frente al publicado en npmjs.org como medida adicional. https://github.com/cheeriojs/cheerio (README oficial), https://socket.dev/npm/package/cheerio, https://security.snyk.io/package/npm/cheerio, https://www.npmjs.com/package/cheerio Restringir acceso del componente mediante configuracion del entorno de ejecución.
